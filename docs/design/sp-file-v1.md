# v1 文件类剪贴板需求优化版

## Summary
- 目标：在现有“纯文本剪贴板”基础上，新增 `文件剪贴项`，支持桌面端和 Web 端发起、查看、同步、接收单个文件。
- v1 边界：`单文件`、`元数据先同步`、`手动点击接收`、`在线直传`、`失败可重试`，不做多文件、文件夹、断点续传、离线可取。
- 成功标准：
  - 发送后，接收端无需先拿到文件内容，就能看到文件图标、名称、大小；图片文件还能看到缩略图。
  - 接收端点击“接收”后，才开始真正传文件内容，并在剪贴项底部看到进度条和百分比。
  - 桌面端接收完成后：保存到默认目录，并写回系统剪贴板。
  - Web 端接收完成后：浏览器下载成功即可，不要求写回系统剪贴板。

## 产品行为
- 剪贴项类型：
  - 保留现有 `文本剪贴项`。
  - 新增 `文件剪贴项`，每条剪贴项只代表一个文件。
- 文件来源：
  - 桌面端：从系统剪贴板复制文件进入应用。
  - Web 端：支持 `手动选择文件`、`粘贴文件`、`拖拽文件` 三种进入方式。
- 列表展示：
  - 非图片文件：展示文件类型图标、文件名、大小（MB）。
  - 图片文件：展示缩略图，同时保留文件名、大小（MB）。
  - 文件剪贴项需有明确状态：`仅元数据`、`接收中`、`已接收`、`接收失败`。
  - 在 `仅元数据` 状态下展示“接收”图标；点击后进入接收流程。
  - 在 `接收中` 状态下，剪贴项底部展示进度条和百分比。
- 同步规则：
  - 文件同步分两段：
    - 第 1 段同步元数据：文件名、扩展名/MIME、大小、类型图标信息、图片缩略图。
    - 第 2 段同步真实文件内容：仅在接收端点击“接收”后开始。
  - 图片缩略图属于元数据的一部分，接收端在未拿到原文件前也能预览。
  - 发送端必须在线且源文件可读；桌面端需保持应用可用，Web 端需保持页面打开。
- 接收结果：
  - 桌面端：保存到默认目录，若重名自动重命名；完成后把该文件写回系统剪贴板。
  - Web 端：触发浏览器下载；完成标准是下载成功，不承诺写回系统剪贴板。
- v1 不包含：
  - 多文件合并成一个剪贴项
  - 文件夹复制
  - 自动预取
  - 暂停/恢复/断点续传
  - 离线中转或发送端离线后继续下载

## 接口与数据模型调整
- 剪贴板项模型需从“文本专用”升级为“类型化剪贴项”：
  - 增加 `itemKind: text | file`
  - 文件项增加 `fileMeta`：
    - `fileName`
    - `extension`
    - `mimeType`
    - `sizeBytes`
    - `thumbnail`（仅图片文件，可为空）
    - `transferState`
    - `progressPercent`
    - `localPath/downloadedAt`（按端区分）
- 同步协议需从“直接传 raw content”升级为“先发剪贴项元数据，再按需拉文件内容”：
  - 现有同步入口应承载 `text/file` 两类剪贴项
  - 新增“接收文件内容”动作接口
  - 新增进度事件，供桌面端/Wails 与 Web/SSE 刷新进度条和百分比
- 前端列表/详情组件要支持混合渲染：
  - 文本项按现有样式
  - 文件项按图标/缩略图/名称/大小/状态/进度渲染


## Assumptions
- 默认目录：
  - 桌面端默认保存到用户下载目录下的 `LocalShareGo` 子目录。
  - Web 端使用浏览器默认下载目录。
- 缩略图策略：
  - 仅图片文件生成并同步压缩缩略图；原图内容仍走手动接收。
- 兼容策略：
  - 后端可以保留现有文本接口语义，但实现上需扩展为同时支持文本和文件项。
- 交互默认：
  - v1 只要求“重试”，不要求“取消接收”与“恢复接收”。
